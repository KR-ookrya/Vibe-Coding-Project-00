## 1. 기능 요구사항 (Functional Requirements)

### 1-1. 입력

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| F-01 | 사용자는 자기소개서 본문을 textarea에 직접 입력하거나 붙여넣기할 수 있다 | Must |
| F-02 | 자기소개서는 최소 50자 이상이어야 분석을 요청할 수 있다 | Must |
| F-03 | 지원 직무를 선택적으로 입력할 수 있다 (미입력 시 분석에서 제외) | Should |
| F-04 | 지원 회사를 선택적으로 입력할 수 있다 (미입력 시 분석에서 제외) | Should |
| F-05 | textarea 하단에 현재 입력 글자 수를 실시간으로 표시한다 | Should |
| F-06 | `Ctrl+Enter` / `Cmd+Enter` 단축키로 분석을 시작할 수 있다 | Could |

### 1-2. 분석 & AI 처리

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| F-07 | 분석 요청 시 Claude Opus 4.7 API를 호출하여 첨삭 피드백을 생성한다 | Must |
| F-08 | 지원 직무·회사가 입력된 경우 프롬프트에 포함하여 맞춤 분석한다 | Must |
| F-09 | API 응답은 SSE(Server-Sent Events)로 스트리밍하여 실시간 출력한다 | Must |
| F-10 | 분석 중에는 버튼을 비활성화하고 로딩 상태를 표시한다 | Must |

### 1-3. 출력 (피드백 구조)

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| F-11 | 종합 평가: 전반적인 수준을 3~4문장으로 제공한다 | Must |
| F-12 | 점수: 100점 만점, 4개 항목으로 세분화하여 제공한다 | Must |
| F-13 | 점수 항목: 내용의 구체성(25점), 문장 표현력(25점), 직무 적합성(25점), 차별화 포인트(25점) | Must |
| F-14 | 잘된 점: 긍정적 요소를 불릿으로 나열한다 | Must |
| F-15 | 개선이 필요한 부분: 문제점과 개선 방법을 구체적으로 설명한다 | Must |
| F-16 | 첨삭된 자기소개서: 수치·사례·성과 중심으로 재작성한 완성본을 제공한다 | Must |
| F-17 | 피드백 결과는 Markdown으로 렌더링하여 가독성 높게 표시한다 | Must |

### 1-4. 결과 활용

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| F-18 | 결과 전체를 클립보드로 복사할 수 있다 | Should |
| F-19 | 분석 완료 후 버튼 텍스트가 "다시 첨삭하기"로 변경된다 | Should |
| F-20 | 분석 시작 시 결과 영역으로 자동 스크롤된다 | Should |

## 2. 비기능 요구사항 (Non-Functional Requirements)

### 2-1. 성능

| ID | 요구사항 |
|----|----------|
| NF-01 | 분석 버튼 클릭 후 3초 이내에 첫 번째 스트리밍 청크가 화면에 출력되어야 한다 |
| NF-02 | 요청 본문 크기는 50KB로 제한한다 (과도한 입력 방지) |
| NF-03 | 전체 첨삭 결과는 30초 이내에 완료되어야 한다 |

### 2-2. 신뢰성 & 오류 처리

| ID | 요구사항 |
|----|----------|
| NF-04 | API 키 인증 실패(401) 시 사용자에게 명확한 안내 메시지를 표시한다 |
| NF-05 | 그 외 API 오류 발생 시 오류 메시지를 결과 영역에 표시하고 버튼을 복구한다 |
| NF-06 | 50자 미만 입력 시 API 호출 전에 클라이언트·서버 양측에서 유효성 검사를 수행한다 |

### 2-3. 보안

| ID | 요구사항 |
|----|----------|
| NF-07 | Anthropic API 키는 `.env` 파일에만 저장하고 클라이언트에 절대 노출하지 않는다 |
| NF-08 | 요청 본문 크기 제한(50KB)으로 과도한 페이로드 공격을 방어한다 |

### 2-4. 사용성

| ID | 요구사항 |
|----|----------|
| NF-09 | 회원가입·로그인 없이 즉시 사용 가능해야 한다 |
| NF-10 | 최신 Chrome / Edge / Safari / Firefox 브라우저에서 정상 동작해야 한다 |
| NF-11 | 한국어 UI로 제공하며, 모든 안내 문구는 한국어로 작성한다 |

## 3. 시스템 제약사항 (Constraints)

| 항목 | 내용 |
|------|------|
| AI 모델 | Claude Opus 4.7 고정 (`claude-opus-4-7`) |
| 최대 응답 토큰 | 4,096 tokens |
| 입력 최대 크기 | 50KB |
| 서버 포트 | 기본 3000 (환경 변수 `PORT`로 변경 가능) |
| 런타임 | Node.js + Express |
| 외부 의존성 | `@anthropic-ai/sdk`, `express`, `dotenv`, `marked` (CDN) |

## 4. 제외 범위 (Out of Scope — v1.0)

- 사용자 계정 / 로그인 / 히스토리 저장
- 데이터베이스 연동
- 공고문 URL 크롤링
- 항목별(성장 과정 / 지원 동기 등) 분리 첨삭
- 모바일 네이티브 앱
- 영문 Cover Letter 지원
- 사용량 분석 / 대시보드

## 5. 완료 기준 (Definition of Done)

- [ ] 자기소개서 입력 → 분석 → 스트리밍 결과 출력 흐름이 오류 없이 동작한다
- [ ] 직무·회사 입력 여부에 따라 프롬프트가 달라진다
- [ ] 4개 항목 점수가 매번 일관된 형식으로 출력된다
- [ ] 클립보드 복사가 정상 동작한다
- [ ] API 오류 시 사용자가 이해할 수 있는 메시지가 표시된다
- [ ] 50자 미만 입력 시 분석이 차단된다
