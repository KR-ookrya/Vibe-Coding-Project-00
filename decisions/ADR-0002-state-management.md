# ADR-0002 — 상태 관리 방식 선택

## 배경

첨삭 결과, 점수, 입력값, 공유 링크 등 여러 UI 상태를 관리해야 했다.
별도 상태 관리 라이브러리(Redux, Zustand 등)를 도입할지 결정이 필요했다.

## 선택지

| 선택지 | 설명 |
|---|---|
| **Vanilla JS (DOM 직접 제어)** | 라이브러리 없이 JS 변수와 DOM으로 상태 관리 |
| Redux / Zustand | 전역 상태 저장소, 복잡한 상태 흐름에 유리 |
| React useState / useReducer | 컴포넌트 기반 상태 관리 |

## 결정

**Vanilla JS — DOM 직접 제어 방식**

## 이유

- 앱의 상태가 단순: 입력값, 스트리밍 결과, 로딩 여부, 점수 정도
- 외부 라이브러리 없이 `innerHTML`, `textContent`, `classList` 로 충분히 처리 가능
- 상태 관리 라이브러리 도입 시 오히려 설정 비용이 더 큼

## 결과

- 의존성 추가 없이 경량 유지
- SSE 스트리밍 수신 중 DOM을 직접 업데이트해 실시간 출력 구현
- Rate limit 초과, API 오류 등 예외 상태도 단순 조건문으로 처리
